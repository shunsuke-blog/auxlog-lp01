# LP定義書 - Auxlog

## 1. LP概要

### 目的
- Auxlogを知ったユーザーが「試してみたい」と思い、`auxlog.app` に遷移してサインアップする
- プライバシーポリシー・利用規約の公開場所としても機能する

### URL
- `https://auxlog.com`

### 技術スタック
- Next.js（auxlog-lpリポジトリ、独立したプロジェクト）
- Vercel（auxlog.comに紐づけ）
- Tailwind CSS

---

## 2. ターゲットユーザー

- **年齢**: 20代後半〜30代
- **属性**: 筋トレ中級者・上級者（筋トレ歴1年以上）
- **特徴**:
  - ジムに週2〜4回通っている社会人
  - やる種目は決まっているが、セット設計が面倒または伸び悩んでいる
  - 感覚に頼ったトレーニングに限界を感じている
  - 朝や仕事終わりのジム前に「今日何をやるか」を考えたくない

---

## 3. デザイン方針

### トーン
- **エネルギッシュ・ボールド・ブラック**
- 黒背景をベースに、力強いタイポグラフィで訴求する
- 余白は広く取りつつ、インパクトのあるビジュアルで引き込む

### カラーパレット
| 用途 | カラー |
|------|--------|
| 背景 | `#0A0A0A`（ほぼ黒） |
| テキスト | `#FFFFFF` |
| アクセント | `#E8FF00`（イエロー・グリーン）|
| サブテキスト | `#888888` |
| カード背景 | `#141414` |
| ボーダー | `#222222` |

### タイポグラフィ
- 見出し: 極太（font-weight: 900）、大きめのサイズ
- 本文: Regular〜Medium
- フォント: `Inter` または `Noto Sans JP`

### UIの参考イメージ
- Apple系のミニマル × スポーツブランド（Nike）のボールド
- 「Appleの純正アプリと並べて恥ずかしくないか」を基準にする

## 5. 各セクション詳細

### 5.1 Hero（ファーストビュー）

**目的**: 一瞬で「これは自分のためのアプリだ」と感じさせる

**構成**
- ロゴ（左上）+ ナビゲーション（右上：「無料で始める」ボタン）
- メインキャッチコピー（大見出し）
- サブコピー
- CTAボタン
- アプリのスクリーンショットまたはモックアップ画像

**コピー案**
```
メインキャッチ:
今日のトレーニングを、
30秒で決める。

サブコピー:
感覚に頼ったトレーニングをやめろ。
Auxlogが前回の記録から、今日やるべき重量・セット・回数を自動で導き出す。

CTAボタン: 30日間無料で始める →
```

---

### 5.2 Problem（課題提起）

**目的**: ターゲットユーザーの「あるある」を言語化して共感を得る

**構成**
- セクションタイトル
- 3つの課題カード

**コピー案**
```
セクションタイトル:
こんな経験、ないか？

課題1:
ジムに着いてから
「今日何やろう」と悩む

課題2:
感覚でやるから
いつもベンチプレスばかり

課題3:
前回何キロ上げたか
覚えていない
```

---

### 5.3 Solution（解決策）

**目的**: Auxlogがその課題を解決することを伝える

**構成**
- セクションタイトル
- 解決策の説明（1〜2文）
- イメージビジュアル

**コピー案**
```
セクションタイトル:
Auxlogが、考える時間をゼロにする。

説明:
前回の記録・疲労度・経過日数から、
今日やるべきメニューを自動で計算。
アプリを開いた瞬間に答えが出ている。
```

---

### 5.4 Features（機能紹介）

**目的**: 具体的な機能を伝えて「本当に使えそう」と思わせる

**構成**
- 3つの機能カード（アイコン＋タイトル＋説明）

**機能カード**
```
機能1: 自動メニュー提案
アイコン: ⚡
タイトル: 開いた瞬間に今日のメニューが出る
説明: 前回の記録をもとに、重量・セット数・回数を自動で提案。悩む時間はゼロ。

機能2: 科学的な重量設計
アイコン: 📈
タイトル: 感覚ではなく、データで伸ばす
説明: RIR（余裕度）と週ボリュームに基づいたプログレッシブオーバーロードで、確実に成長できる重量を提案。

機能3: シンプルな記録
アイコン: ✓
タイトル: 30秒で記録完了
説明: 提案値が初期値として入力済み。変更がなければそのまま保存するだけ。
```

---

### 5.5 How it works（使い方）

**目的**: 使い始めるまでの流れを伝えてハードルを下げる

**構成**
- 3ステップ

**コピー案**
```
STEP 1: 種目を選ぶ
ベンチプレス、スクワットなど、自分がやる種目をタップで選択。

STEP 2: 毎朝アプリを開く
今日やるべき種目・重量・セットが自動で表示される。

STEP 3: 記録して成長する
トレーニング後に結果を入力。次回への提案がさらに精度を上げる。
```

## 6. プライバシーポリシー・利用規約

### ページURL
- `https://auxlog.com/privacy` - プライバシーポリシー
- `https://auxlog.com/terms` - 利用規約

### 記載必須項目（プライバシーポリシー）
- 収集する個人情報の種類（メールアドレス、トレーニングデータ）
- 情報の利用目的
- 第三者提供（Supabase、Stripe、Vercel）
- Cookie・解析ツールの使用
- お問い合わせ先
- 運営者情報

### 記載必須項目（利用規約）
- サービス概要
- 利用料金・支払い
- 禁止事項
- 免責事項
- サービスの変更・終了
- 準拠法・管轄裁判所

## 10. 実装上の注意事項

- `auxlog-lp` として独立したNext.jsリポジトリで管理
- `auxlog.com` をVercelの別プロジェクトとして紐づける
- アプリ本体（auxlog.app）とは完全に分離
- プライバシーポリシー・利用規約は静的ページとして実装
- Google Analytics または Vercel Analyticsでアクセス計測
